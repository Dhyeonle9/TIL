# 1회차 강의 내용 요약

1. github.io 페이지 작성

2. 협업시 git 이용
    - 하나의 branch 안에서 같은 repository 이용하여 clone/push/pull (충돌 위험 큼)
        - 같은 라인을 동시 수정하여 업데이트하게되면 충돌이 발생하며, 다시 pull하여 merging과정을 거쳐야한다.
    - branch 활용 (이상적)
        - `git branch`
        - `git branch -c <브랜치>`
        - `git switch <브랜치>`
    
        - github를 이용한 코드 merging
